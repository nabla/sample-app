Create an OAuth Client

Generate an RSA key pair, then upload the public key to the API Admin Console when creating your OAuth Client (choose JWT Public Key).

Private key stays on the server — never leaves your machine.

Paste this public key in the Admin Console. Regenerating will break any existing OAuth client using the old key.

Generate a Server Access Token

The backend constructs a JWT client assertion signed with your private key and exchanges it for a server access token.

POST /oauth/token
{
"grant_type": "client_credentials",
"client_assertion_type": "urn:ietf:params:oauth:client-assertion-type:jwt-bearer",
"client_assertion": "<signed JWT>"
}
✓ Server token generated — stored by the server only

The server access token never reaches the browser. The backend keeps it and uses it to provision users.

Create & Authenticate a User

Creates a Nabla user, then exchanges the server token for a user access token. The user is only created once — re-running this step re-authenticates the same user.

1. POST /v1/core/server/users
Authorization: Bearer <SERVER_ACCESS_TOKEN>
→ returns { id, external_id, … }
2. POST /v1/core/server/jwt/authenticate/{id}
Authorization: Bearer <SERVER_ACCESS_TOKEN>
→ returns { access_token, refresh_token }
✓ User session established
nabla_user_id

The access token (5 min) and refresh token (30 days) are stored in localStorage and will be used to authenticate. The access token is automatically renewed by this application when required.
